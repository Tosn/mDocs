# mDocs · 桌面端文档查看 / 搜索 / AI 问答工具

把分散的文档（Markdown / 文本 / PDF / 网页）集中归纳，**能看、能管、能搜**，并通过 AI **跨文档库对话**直接得到**带来源**的答案。

> **本地优先**：查看、管理、关键词搜索完全离线；仅「AI 问答 / 文本嵌入 / 网页爬取」需要联网。API Key 经操作系统安全区（safeStorage）加密保存，不会明文落库。

---

## 一、安装与启动

```bash
pnpm install      # 安装依赖
pnpm dev          # 启动应用（开发）
pnpm build        # 三端打包
```

> **原生模块双 ABI（重要）**：`better-sqlite3` 在「跑应用（Electron）」与「跑测试（Node）」下 ABI 不同。
> - 跑应用前若启动报 `NODE_MODULE_VERSION` 不匹配：`pnpm rebuild:sqlite`（拉 Electron 预编译）。
> - 跑测试前：`pnpm test`（`pretest` 会自动切回 Node ABI）。

### 下载安装包首次打开（未签名，需手动放行）

GitHub Actions 打出的安装包**未做代码签名 / 公证**，从网上下载后系统会拦截：

**macOS**（拖入「应用程序」后图标带「禁止」标记、双击打不开）：
```bash
# 去掉「隔离」标记
sudo xattr -dr com.apple.quarantine /Applications/mDocs.app
# 若仍打不开（Apple Silicon），再补一个本地 ad-hoc 签名
codesign --force --deep --sign - /Applications/mDocs.app
```
之后正常从启动台/应用程序打开即可。

**Windows**（双击 `.exe` 弹出 SmartScreen 蓝色警告）：点「**更多信息 → 仍要运行**」即可安装，不影响使用。

> 正式对外分发需 Apple 开发者账号做签名+公证 / Windows 代码签名证书；仅自用走上面的手动放行即可。

## 二、界面总览

三栏布局：

- **左栏 · 文档库**：文件夹 + 文件的统一树（类似 VSCode）。顶部「**＋ 新建** / 回收站 / 设置」，底部关键词搜索。
- **中栏 · 阅读区**：打开文档后在此查看 / 编辑（PDF 内嵌预览，md/txt 可编辑保存）。
- **右栏 · AI 对话**：选模型、提问、看带来源的流式回答、管理历史会话。

## 三、文档库：组织与管理

### 文件树
- 「**默认**」节点：未归类文档的默认存放处（不可改名/删除）。
- 文件夹可**展开/收起**，文件为叶子，**点击文件即在中栏打开**。
- 文件名按类型显示后缀并区分图标：**`.md` 蓝色 / `.txt` 灰色 / `.pdf` 红色**。文件夹名后 `(n)` 为其直接包含的文档数。

### 新建 / 上传 / 添加网页
有两种入口（**操作目标 = 你右键的文件夹，或左上＋菜单的「默认」**）：
- **左上「＋ 新建」**：在「默认」下新建文档 / 上传文件 / 添加网页 / 新建文件夹。
- **右键文件夹**：在此新建文档 / 上传到此 / 添加网页到此 / 新建子文件夹 / 重命名 / 删除。
- **右键空白处**：等同于「默认」的创建菜单。

> 新建文档时**按你输入的扩展名定类型**：输入 `note.txt` → txt 文件；`note.md` 或无后缀 → md。

### 改名 / 移动 / 删除
- **改名**：选中文件/文件夹按 **`Enter`**（或 `F2` / 双击名称）进入内联编辑，**再次 `Enter` 或失焦保存**，`Esc` 取消。默认只选中文件名主体（不含后缀）。
- **移动**：把文件**拖到目标文件夹**（拖到空白=移到根/默认），或右键文件操作。
- **删除**：右键文件/文件夹 → 删除。删除项进入**回收站**。

### 回收站（保留 7 天）
顶部「回收站」打开：可**恢复**或**彻底删除**；支持**复选框批量**恢复/彻底删除。超过 7 天的项目在启动时自动清除。

## 四、网页爬取（重点）

「添加网页」对话框里粘贴链接后有**两种抓取方式**：

### 1) 「添加」——普通抓取（直接请求）
直接请求网页 HTML，用 Readability 抽正文、转成 Markdown 入库（保留图片、表格）。
适合**公开、无需登录、反爬不严**的页面（博客、文档站、大多数资讯页）。

### 2) 「需要登录？用浏览器打开后抓取」——**网页渲染抓取（强烈推荐用于反爬站点）**
点这个会**弹出一个内置浏览器窗口**加载该链接，你可以在里面**正常登录、滑动验证、跳转到目标页**，确认页面加载完整后点「**抓取当前页**」——程序抓取的是你**当前已渲染/已登录**页面的 HTML，再抽正文入库。

> **知乎 / 微信公众号 / 需要登录或有强反爬的站点，请优先用这种「渲染抓取」**：
> - 这类站点对「直接请求」常返回 **403 / 需要登录 / 空白**（普通抓取会失败并提示）。
> - 渲染抓取走的是真实浏览器环境（带你的登录态、执行了 JS），能拿到完整正文，规避大部分反爬。

**使用建议**：
1. 先试「添加」（快）；若提示 `HTTP 401/403`、需要登录或正文为空 →
2. 改用「需要登录？用浏览器打开后抓取」，在弹窗里登录/打开到正文页 → 点「抓取当前页」。

> 抓取的网页文档正文是 Markdown，在文件树中显示为 **`.md`**。

## 六、AI 问答（跨文档库，带来源）

### 1) 先在「设置」里配置模型（两类，各自独立 Key）
- **对话模型**（生成回答）：OpenAI GPT-4o / Anthropic Claude 3.5 / **DeepSeek Chat** / 通义千问 / **智谱 GLM-4** 等。
- **嵌入模型**（把文档/问题转向量做检索，**RAG 必需**）：OpenAI Embedding 3 / 通义千问 Embedding v3 / **智谱 Embedding-3**。

> **重要**：DeepSeek、Anthropic **没有嵌入模型**。若对话用它们，嵌入模型请另选 **OpenAI / 通义 / 智谱** 之一，并填**对应厂商**的 Key。
> 每个模型的 Key 独立保存；切换到已配置过的模型会以**密文**回显，可直接沿用或输入新值更新。

配好「嵌入模型」并保存后会**自动重建一次文档索引**（也可在设置里点「重建文档索引」手动重建，导入大量文档或换嵌入模型后建议手动重建）。

### 2) 提问
- 在右栏输入框提问，回车发送（`Shift+Enter` 换行）。
- 输入 **`@`** 可**限定范围**到某些文件/文件夹（只在所选范围内检索）。
- 回答**流式输出**，先显示「思考中…」，随后逐字呈现；下方列出**来源文档标题**（点击可跳转打开，悬停看命中片段）。
- 找不到相关内容时明确回答「未在文档中找到相关内容」，且不附噪声来源。

### 3) 历史会话
右栏顶部：「**＋ 新会话**」开新对话；下拉**切换历史会话**（以首条提问为标题）；「**删除**」删除当前会话。空会话不会堆积。

### 常见报错排查
- `嵌入请求失败 / 对话请求失败 HTTP 401`：对应模型的 **API Key 无效**（确认填的是完整 Key、账户有额度）。
- `嵌入调用失败：fetch failed`：网络不可达或端点不对（已对各厂商自动用其官方 baseUrl；若仍失败检查代理/网络）。
- 提问一直「未找到相关内容」：通常是**还没配置可用的嵌入模型**或**索引未建**——配好嵌入模型并重建索引。

## 七、隐私与安全
- API Key 经 **OS safeStorage 加密**存于本机安全区，不明文入库、不上传。
- 仅**问答、嵌入、网页爬取**会联网（前两者把相关文档片段发送给你**自选**的模型厂商）。
- 查看、管理、关键词搜索、回收站均**离线**可用。
